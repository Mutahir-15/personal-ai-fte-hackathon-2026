# Data Model: Silver Tier Communication

## Entities

### EmailAction
- **Source**: Detected by `gmail_watcher.py` or drafted by `email-mcp`.
- **Fields**:
  - `email_id`: Unique Gmail message ID.
  - `thread_id`: Gmail thread ID for conversation context.
  - `sender`: From address and name.
  - `recipient`: To address.
  - `subject`: Email subject.
  - `snippet`: First 500 characters of body.
  - `received_at`: ISO 8601 timestamp.
  - `priority`: `high` | `medium` | `low` (based on label/keywords).
  - `is_new_contact`: Boolean (checked against `TRUSTED_CONTACTS`).

### WhatsAppAction
- **Source**: Detected by `whatsapp_watcher.py`.
- **Fields**:
  - `message_hash`: SHA256 of `contact + timestamp + preview`.
  - `contact_name`: WhatsApp display name.
  - `preview_text`: Message preview (max 500 chars).
  - `keyword_matched`: Which priority keyword triggered detection.
  - `detected_at`: ISO 8601 timestamp.

### LinkedInPost
- **Source**: Generated by `linkedin_poster.py` via Gemini CLI.
- **Fields**:
  - `post_id`: Internal UUID.
  - `headline`: Post headline (max 150 chars).
  - `body`: Post content (max 1300 chars).
  - `hashtags`: List of 5-8 hashtags.
  - `status`: `pending_approval` | `approved` | `queued` | `published` | `failed`.
  - `scheduled_for`: ISO 8601 timestamp or null.

### ApprovalRequest (Enhanced)
- **Source**: Created by any Silver action requiring HITL.
- **Fields**:
  - `approval_id`: UUID.
  - `type`: `email_send` | `email_reply` | `linkedin_post` | `whatsapp_reply`.
  - `target`: Recipient email or platform name.
  - `priority`: `critical` | `high` | `medium` | `low`.
  - `created_at`: ISO 8601 timestamp.
  - `expires_at`: `created_at` + 24 hours.
  - `status`: `pending` | `approved` | `rejected` | `expired`.
  - `payload`: JSON object containing the action-specific data (e.g., email body, post text).

## State Transitions (Approval)
1. **Pending**: Request created in `/Pending_Approval/`.
2. **Approved**: File moved to `/Approved/`. `silver_hitl_approval.py` detects, updates `approved_actions.json`, and archives.
3. **Rejected**: File moved to `/Rejected/`. `silver_hitl_approval.py` detects and archives.
4. **Expired**: Expiry check thread moves file to `/Done/expired/`.
